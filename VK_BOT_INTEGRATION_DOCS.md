# Документация по Интеграции VK Бота и Программы Лояльности

## Описание
В рамках данного обновления была полностью реализована интеграция с ВКонтакте для автоматического начисления виртуальных бонусных баллов за активность пользователей в группе (лайки, комментарии) и связывания профиля клиента с CRM (iiko) при получении номера заказа в личные сообщения сообщества. Оповещения об изменении статуса заказа теперь также отправляются в личные сообщения ВКонтакте.

## Реализованный Функционал

### 1. База данных и Модели
Были добавлены следующие сущности в БД (и созданы соответствующие Alembic миграции):
- `VkUser` — профиль пользователя ВК. Хранит `vk_id`, привязанный номер телефона, CRM `customer_id` и локальный баланс баллов за активность.
- `VkActivity` — лог активности пользователя (например, лайки, комментарии на стене). Защищает от повторного начисления баллов за одно и то же действие.
- `VkSettings` — таблица настроек, позволяющая управлять токенами ВКонтакте без редактирования файла `.env`.

### 2. Админ-Панель (Admin Panel)
Параметры интеграции теперь можно настраивать прямо из интерфейса админ-панели:
- **Путь:** Раздел `Система` -> `Настройки VK Бота` (`/settings/vk`).
- **Стек:** Vue.js + Vuetify + FastAPI.
- Форма предоставляет поля для ввода **VK Bot Token**, **Кода подтверждения** (для Callback API) и **Секретного ключа**. Ключи сохраняются в таблицу `vk_settings` и применяются моментально без перезагрузки бэкенд-сервера.

### 3. Обработка Webhook (Backend)
Реализован эндпоинт `POST /api/v1/vk/webhook`, который ВКонтакте вызывает при наступлении событий в сообществе:
- **Подтверждение сервера:** Возвращает код подтверждения из `VkSettings`.
- **Новое сообщение (`message_new`):** Если пользователь пишет сообщение, содержащее ключевое слово (например, "Заказ №12345"), бот автоматически ищет заказ в БД, достаёт номер телефона и привязывает профиль VK к аккаунту гостя в iikoCard.
- **Активность (`like_add`, `wall_reply_new`):** Если пользователь привязан, бот начисляет локальные баллы (например, 5 за лайк, 10 за комментарий) в профиль `VkUser` (таблица `vk_users` поле `vk_bonus_balance`).

### 4. Уведомления о статусах заказов
В сервисе синхронизации `iiko_sync_service.py` (метод `process_iiko_order`) добавлена проверка изменения статуса заказа.
- Если статус меняется (например, заказ переходит в статус "Готовится" или "В пути"), система находит `VkUser` по номеру телефона гостя и отправляет персонализированное сообщение со статусом (и именем курьера, если доступно).

### 5. Синхронизация баллов в iikoCard
Создан эндпоинт `POST /api/v1/iiko/sync-vk-loyalty`, который переводит накопленные баллы активности VK из локальной БД на кошелек клиента в системе лояльности iikoCard.
- Бэкенд ищет всех `VkActivity` с флагом `is_synced = False`.
- Группирует баллы по пользователям, использует `iiko_service` для поиска профиля по телефону.
- Начисляет баллы на виртуальный кошелек пользователя в iiko.
- Помечает записи синхронизированными и уведомляет клиента в ВКонтакте.

## Как запустить и протестировать

1. Откройте Админ-Панель: `http://localhost:8081` 
   - Перейдите в **Система > Настройки VK Бота**. Укажите ключи вашей группы ВКонтакте.
2. Подключите Webhook в VK: 
   - Зайдите в управление вашим сообществом VK -> API -> Callback API.
   - Впишите туда ваш URL (например, сгенерированный ngrok, оканчивающийся на `/api/v1/vk/webhook`).
   - Нажмите "Подтвердить" (бэкенд должен вернуть тот же код подтверждения, что вы указали в админке).
3. Привязка пользователей: 
   - Напишите в сообщения сообщества сообщение: `Заказ №[номер существующего заказа]`. Вы должны получить сообщение об успешной интеграции.
4. Тест синхронизации:
   - Сделайте POST-запрос на `http://localhost:8000/api/v1/iiko/sync-vk-loyalty` из Postman / Swagger для перевода локальных баллов VK в iikoCard.
