# SQL FAQ - Учет полей базы данных (Клиенты и Лояльность)

Этот документ содержит описание полей и таблиц базы данных, связанных с клиентами и программой лояльности.
Дата последнего обновления: 27.04.2026 14:30 (Добавлен функционал нормализации и объединения телефонов)

## Таблица `customers`

| Поле                         | Название         | За что отвечает        | Пояснения                                                    |
| :--------------------------- | :--------------- | :--------------------- | :----------------------------------------------------------- |
| `id`                         | ID               | Первичный ключ         | Автоинкремент                                                |
| `phone`                      | Телефон          | Основной номер         | Уникальный. Формат: `+79991234567` (строго) |
| `name`                       | Имя              | Имя гостя              |                                                              |
| `surname`                    | Фамилия          | Фамилия гостя          | Обновляется при каждом заказе, если передана                 |
| `email`                      | Email            | Почта                  |                                                              |
| `birthday`                   | День рождения    | Дата рождения          | Используется для акций                                       |
| `bonus_points`               | Бонусы           | Баланс баллов          | Decimal(10,2)                                                |
| `iiko_customer_id`           | iiko ID          | ID в iiko Loyalty      | UUID из iiko                                                 |
| `notes`                      | Заметки          | Системные заметки      | История заказов (Номер, Дата, Сумма, Состав)                 |
| `iiko_comment`               | iiko Коммент     | Комментарий из iiko    | Используется для парсинга доп. телефонов                     |
| `registration_date`          | Дата рег.        | Дата создания в iiko   |                                                              |
| `last_order_date`            | Посл. заказ      | Дата последнего заказа | Обновляется при каждом заказе                                |
| `total_orders_count`         | Кол-во заказов   | Всего заказов          | Из OLAP iiko (Sales)                                        |
| `total_orders_amount`        | Сумма заказов    | Общая сумма выкупа     | Из OLAP iiko (Sales) - LTV                                   |
| `is_new_guest`               | Новый гость      | Флаг новичка           | Снимается при закрытии первого заказа                        |
| `is_high_risk`               | Группа риска     | Флаг высокого риска    | Определяется по категориям iiko                              |
| `risk_reason`                | Причина риска    | Описание причины       | Категория iiko с ключевым словом                             |
| `iiko_notes`                 | Заметки iiko     | Комментарий гостя      | Синхронизируется с iiko Loyalty                              |
| `total_purchases_sum`        | Сумма покупок    | Сумма всех оплат       | Поле для Laravel админки                                     |
| `last_iiko_order_id`         | Посл. iiko заказ | UUID заказа iiko       | Мапится из `UniqOrderId` (OLAP) или `OrderId`                |
| `orders_history`             | Ист. заказов     | JSON история из iiko   | **[UPD 27.04]** JSONB кэш заказов. Содержит: `id`, `number`, `date`, `sum`, `status`, `exists_locally`, `details` (items). |
| `should_be_checked_for_risk` | Флаг риска iiko  | Статус риска из iiko   | **[NEW]** Прямой флаг из iiko Cloud API (Loyalty)            |
| `first_name`                 | Имя (iiko)       | Очищенное имя          | **[NEW]** Для корректного обращения (из Cloud API)           |
| `last_name`                  | Фамилия (iiko)   | Очищенная фамилия      | **[NEW]** Из Cloud API                                       |
| `uid`                        | UUID iiko        | Уникальный ID          | **[NEW]** Основной ID в iiko Transport                       |
| `iiko_id`                    | iiko ID          | Альтернативный ID      | **[NEW]** Совпадает с iiko_customer_id                       |
| `iiko_categories`            | Категории iiko   | Список категорий       | **[NEW]** JSON массив имен категорий                         |
| `additional_phones`          | Доп. телефоны    | Список номеров         | **[NEW]** JSON массив дополнительных номеров                 |
| `gender`                    | Пол              | Пол гостя              | **[NEW]** Не указан / Мужской / Женский (из iiko)            |
| `marketing_consents`        | Согласия         | Подписки на рассылки   | **[NEW]** JSONB массив объектов согласий из iiko             |
| `consent_status`            | Статус согласия  | Общий статус           | **[NEW]** Статус согласия на обработку данных                |
| `updated_at`                 | Обновлено        | Время посл. изменения  |                                                              |
