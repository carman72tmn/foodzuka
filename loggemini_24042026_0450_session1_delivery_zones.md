# Лог изменений FoodZuka (Session 1: Delivery Zones)
Дата: 24.04.2026
Время: 04:50 (UTC+5)

## Описание задачи
Реализация автоматического определения зоны доставки по адресу заказа при синхронизации из iiko Cloud.

## Внесенные изменения:
1. **backend/app/services/iiko_service.py**:
   - Добавлен импорт `yandex_service`.
   - Реализован асинхронный метод `check_address_zone(organization_id, city, street, house)`.
   - Логика: Геокодирование адреса -> Получение координат -> Проверка вхождения в полигоны (Point-in-Polygon).
   
2. **backend/app/services/iiko_sync_service.py**:
   - Интегрирован вызов `check_address_zone` в цикле синхронизации заказов.
   - Теперь для заказов типа "Доставка" заполняются поля: `delivery_zone` (текст), `resolved_delivery_zone_id` (FK), `latitude`, `longitude`.

3. **Документация**:
   - `system_faq.md`: Описана архитектура зон и полигонов. Исправлена кодировка файла на UTF-8.
   - `sql_faq.md`: Добавлена информация о таблицах `delivery_zones` и `custom_polygons`.
   - `sitenav.md`: Создана карта функционала админки (Заказы, Филиалы, Клиенты).

## Деплой и проверка:
- Файлы выгружены на VPS `foodtech`.
- Выполнен перезапуск контейнера `backend`.
- Ошибок в логах при запуске не обнаружено.

## Известные ограничения:
- Требуется наличие валидного API-ключа Яндекс.Геокодера в настройках.
- Полигоны должны быть предварительно загружены через KML в админ-панели.
