# Справочник переменных iiko RESTO OLAP (тип SALES)

Ниже приведен список наиболее часто используемых переменных для формирования отчетов через iiko RESTO API v2. Эти поля используются в секциях `columns` (колонки отчета) и `filters` (фильтры).

## 1. Показатели (Measures) — Числовые данные для агрегации

| Поле | Описание (RU) | Тип данных |
| :--- | :--- | :--- |
| `DishSumAfterDiscount.Sum` | **Итоговая выручка** (после всех скидок) | Число |
| `DishAmount` | Количество блюд или проданных единиц | Число |
| `DishCost.Sum` | Себестоимость проданных товаров (общая) | Число |
| `DishMargin.Sum` | Наценка (Выручка - Себестоимость) | Число |
| `DishDiscountSum.Sum` | Общая сумма предоставленных скидок | Число |
| `GuestNum` | Количество гостей в заказах | Число |
| `OrderNum.OrdersCount` | Общее количество заказов | Число |
| `AverageCheck` | Средний чек (вычисляемое поле) | Число |
| `PrepaySum` | Сумма внесенных предоплат | Число |

## 2. Измерения (Dimensions) — Группировка и фильтрация

### Время и Датировка
*   `OpenDate.Typed` — **Учетный день** (рекомендуется для фильтрации по датам).
*   `OpenYear` / `OpenMonth` / `OpenDay` — Год, месяц и день открытия заказа.
*   `OpenDayOfWeek` — День недели (понедельник, вторник и т.д.).
*   `OpenHour` — Час открытия заказа (для почасового анализа).

### Организация и Склад
*   `Department.Name` — Название отделения (терминала).
*   `Store.Name` — Название склада, с которого списаны блюда.
*   `CashServer.Name` — Имя кассового сервера.
*   `Group.Name` — Название группы подразделений.

### Меню и Продукция
*   `DishName` — Наименование блюда (товара).
*   `DishCategory.Name` — Категория блюда (например, "Горячее", "Напитки").
*   `DishGroup.Name` — Группа блюда в меню.
*   `Product.Article` — Артикул товара.

### Персонал и Заказы
*   `Waiter.Name` — Имя официанта/сотрудника, открывшего заказ.
*   `Cashier.Name` — Имя кассира.
*   `OrderNum` — Номер заказа.
*   `OrderType.Name` — Тип заказа (например, "Доставка", "Внутри", "Самовынос").
*   `Table.Number` — Номер стола.

### Оплата
*   `PayTypes` — Типы оплат в заказе.
*   `CardSource.Name` — Источник карты лояльности.

## 3. Как получить актуальный список для вашего сервера?

Список доступных полей может меняться в зависимости от версии iiko Office. Чтобы получить актуальный список именно для вашего сервера, выполните GET запрос:

`GET /resto/api/v2/reports/olap/columns?reportType=SALES`

**Параметр `reportType` может быть:**
- `SALES` (Продажи — основной)
- `DELIVERIES` (Доставки)
- `STOCKS` (Остатки на складах)
- `TRANSACTIONS` (Проводки)

## 4. Особенности фильтрации (v2)

Для фильтрации по датам используйте структуру:
```json
"filters": {
  "OpenDate.Typed": {
    "filterType": "DateRange",
    "periodType": "CUSTOM",
    "from": "YYYY-MM-DD'T'00:00:00.000",
    "to": "YYYY-MM-DD'T'00:00:00.000",
    "includeLow": true,
    "includeHigh": false
  }
}
```
*Важно: Если `includeHigh` равен `false`, то дата `to` не включается в отчет. Поэтому для отчета за один день "Сегодня" ставьте `to` как "Завтра".*
