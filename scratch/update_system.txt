
### Исправление синхронизации отчетов по курьерам (24.04.2026)

Выявлена и исправлена проблема, из-за которой отчеты по курьерам за текущий день не синхронизировались:

1.  **Маппинг курьеров по iiko_id**: Ранее сопоставление курьера из OLAP-отчета с базой данных происходило только по имени. Поскольку в iiko Resto имена в OLAP (например, "Эльнур") могут отличаться от имен в списке сотрудников (например, "Довезука Курьер"), синхронизация пропускала записи.
2.  **Добавление ID в OLAP**: В метод `get_resto_detailed_deliveries` (iiko_service.py) добавлена колонка `Delivery.Courier.Id`.
3.  **Кэширование по ID**: В `sync_courier_deliveries` (iiko_sync_service.py) добавлен кэш курьеров по `iiko_id`, что обеспечивает 100% точность сопоставления.
4.  **Синхронизация имен**: В `get_resto_employees` добавлен парсинг тега `<name>`, который теперь используется как основное имя сотрудника в БД, соответствующее отображаемому имени в iiko.

**Технические детали:**
- Файлы: `backend/app/services/iiko_service.py`, `backend/app/services/iiko_sync_service.py`.
- База данных: Таблица `employees`, поле `name` теперь обновляется из iiko `<name>`.
- Логика: Приоритет сопоставления в OLAP теперь по `Delivery.Courier.Id`.
