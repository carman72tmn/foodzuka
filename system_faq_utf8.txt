### Стабилизация синхронизации выручки (OLAP) (24.04.2026)

Восстановлена и стабилизирована система синхронизации выручки через iiko Server API (OLAP v2).

**Ключевые изменения:**
1.  **Исправление API схем**: В запросах OLAP v2 поле CashRegister заменено на корректное CashRegisterName. Использование неверного имени поля приводило к ошибке 400 Bad Request.
2.  **Группировка по терминалам**: Добавлена детализация по кассовым терминалам (	erminal_name). Теперь выручка не суммируется в одну кучу, а разбивается по конкретным кассам (например, "Атол 20Ф").
3.  **Разбивка по типам оплат**: Интегрирована выгрузка оплат через тот же OLAP-запрос. Добавлены поля `cash_sum` (наличные), `card_sum` (карты) и `online_sum` (онлайн оплаты).
4.  **Устранение циклического импорта**: Исправлена ошибка ImportError в pp/core/datetime_utils.py путем переноса импорта IikoSettings внутрь функций. Это обеспечило стабильную работу фоновых задач.

**Технические детали:**
-   **Файлы**:
    -   ackend/app/services/iiko_service.py: Методы get_olap_report и get_payment_types_report обновлены для работы с CashRegisterName.
    -   ackend/app/services/revenue_sync.py: Логика sync_period обновлена для маппинга оплат по терминалам.
    -   ackend/app/core/datetime_utils.py: Исправлен импорт настроек.
-   **База данных**: Таблица olap_revenue_records расширена новыми полями (см. sql_faq.md).
-   **Периодичность**: Синхронизация запускается каждые 30 минут через фоновые задачи.
